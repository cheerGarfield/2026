 关于坐标：K230 到底发的是什么

  这是需要跟队友说清楚的关键点。看代码中 ALIGN 模式的发送：

  # 320×240 检测 → ×2 → 640×480 发送
  tx_d0 = rect_cx * SCALE   # SCALE=2, 靶心X
  tx_d1 = rect_cy * SCALE   # 靶心Y

  你发送的是靶心在画面中的绝对像素坐标，坐标系是 640×480：

          X=0                              X=639
      Y=0  ┌────────────────────────────────┐
           │                                │
           │          靶心●(D0, D1)          │
           │                                │
           │              ✚ (320, 240)      │  ← 画面中心
           │                                │
   Y=479   └────────────────────────────────┘

  ┌────────────┬────────────────────────────┬────────────┐
  │    字段    │            含义            │  取值范围  │
  ├────────────┼────────────────────────────┼────────────┤
  │ D0 (靶心X) │ 靶心在画面中的水平像素坐标 │ 0 ~ 639    │
  ├────────────┼────────────────────────────┼────────────┤
  │ D1 (靶心Y) │ 靶心在画面中的垂直像素坐标 │ 0 ~ 479    │
  ├────────────┼────────────────────────────┼────────────┤
  │ 画面中心   │ —                          │ (320, 240) │
  └────────────┴────────────────────────────┴────────────┘

  队友做 PID 时需要自己算偏差：

  偏差_X = D0 - 320   // 正值=靶在右边，电机需右转
  偏差_Y = D1 - 240   // 正值=靶在下边，电机需下转

  PID 的目标就是把这两个偏差收敛到 0，即靶心位于画面正中心。

  到 AIM 模式时，除了 D0/D1（靶心），K230 还会发 D2/D3（光斑坐标，同样是 640×480 坐标系）和
  Error（光斑到靶心的欧氏距离，单位像素）。

  总结一句话：你发的不是"偏移量"，而是"靶心在画面中的绝对位置"。队友需要自己拿 (320, 240) 做减法算出偏差，再喂给 PID。
